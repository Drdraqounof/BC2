The Campaigns feature in EduPanel is designed to help teachers take structured, goal-oriented actions to improve student outcomes. Instead of acting as a generic analytics dashboard, Campaigns provide a focused system where teachers can identify problems, take action, and track progress over time.
This feature transforms raw classroom data into meaningful initiatives, allowing teachers to move from observation to intervention.

Purpose
Traditional dashboards often present data without clear direction. The Campaigns feature solves this by answering three key questions:
What is the problem?
What action should be taken?
Is progress being made?
By doing this, EduPanel shifts from passive tracking to active classroom management.

Core Concept
A Campaign represents a targeted effort to improve a specific area of classroom performance.
Examples of Campaigns:
Missing Assignments Recovery
Attendance Improvement
Test Preparation Initiative
Student Performance Boost
Parent Engagement Push
Each campaign is tied to:
A goal
A group of students
Measurable progress

Key Features
1. Active Campaigns Section
Displays all ongoing campaigns in a clear, card-based layout.
Each campaign includes:
Campaign title
Number of students affected
Defined goal
Current status (e.g., In Progress, Completed)
Quick action buttons

2. Campaign Cards
Each campaign is presented as an interactive card.
Example:
Missing Assignments Recovery
8 students affected
Goal: Reduce to 2 by Friday
Status: In Progress
Actions:
View Students
Send Reminder

3. Create Campaign Functionality
Teachers can create new campaigns based on classroom needs.
Workflow:
Select campaign type (attendance, grades, etc.)
Define goal (e.g., reduce missing work)
Select students
Set timeframe
Launch campaign
This enables teachers to quickly respond to issues as they arise.

4. Insights Section
Provides actionable data that can trigger campaigns.
Examples:
Students with missing assignments
Students below attendance threshold
At-risk students
This section helps teachers identify where intervention is needed.

5. Progress Tracking
Tracks improvement over time for each campaign.
Examples:
Missing assignments decreasing week by week
Attendance rates improving
Grade averages increasing
This allows teachers to measure the effectiveness of their actions.

UI/UX Structure
Recommended Layout
Page Title: Campaigns
Create Campaign Button
Active Campaigns Section (cards)
Insights Section (problem indicators)
Optional Progress Visualization (charts)

Design Principles
1. Action-Oriented
Every element should guide the teacher toward a decision or action.
2. Clarity Over Complexity
Avoid overwhelming charts or unnecessary data. Focus on what matters.
3. Contextual Data
Data should always answer “why it matters” and “what to do next.”
4. Simplicity
Teachers should be able to understand and act within seconds.

Improvements Over Current Design
Current Issues:
Generic “Campaign Overview” lacks meaning
Charts do not provide actionable insights
No clear connection between data and action
Recommended Changes:
Replace “Campaign Overview” with “Active Campaigns”
Use campaign cards instead of generic charts
Add actionable insights and quick actions
Introduce campaign creation flow

Key Value
The Campaigns feature provides:
Structured classroom interventions
Clear visibility into student issues
Measurable progress tracking
Reduced reliance on manual tracking

Conclusion
The Campaigns feature is a core differentiator for EduPanel. It elevates the platform from a simple management tool to a decision making system for teachers.
By focusing on goals, actions, and outcomes, EduPanel empowers teachers to actively improve their classroom performance rather than just monitor it.
